# Альфа-Банк - сервис обмена районами

Интерактивный PWA-макет сервиса «Карта обмена районами» для сотрудников доставки банковских продуктов.

## Что реализовано

- мобильный экран с интерактивной картой Санкт-Петербурга;
- стартовая карта без фейковых пользователей;
- точки сотрудников появляются только после добавления пользователем;
- кластеризация точек при отдалении и разделение при приближении;
- ручная постановка своей точки тапом по карте;
- добавление или перенос точки через геолокацию;
- карточка пользователя с ФИО, телефоном, Telegram, MAX, локацией, комментарием и статусом;
- действия: позвонить, написать в Telegram, написать в MAX, предложить обмен;
- создание, редактирование, перенос и удаление своей точки;
- ограничение «одна активная точка с устройства» через `device_id` в `localStorage` и cookie;
- предупреждение о публичности данных перед публикацией;
- входящие и исходящие предложения обмена внутри сайта;
- принятие и отказ от обмена;
- автоматическая смена координат пользователей при принятом обмене;
- таймер автоочистки в 23:59 по часовому поясу Санкт-Петербурга;
- PWA-манифест и service worker;
- переключатель темной и светлой темы;
- мобильная компоновка без выбора города в интерфейсе.

## Текущий backend-режим

Сайт работает без регистрации. Фронт публикуется на GitHub Pages (`https://goswitch.ru/`), а backend запускается на этом ПК. Активные точки и заявки на обмен хранятся на локальном Node.js-сервере в файле `data/changeplace-data.json`. Создание, редактирование и удаление своей точки привязаны к `device_id`, который сохраняется в `localStorage` и cookie браузера.

Если сервер недоступен, приложение остаётся локальным макетом: данные сохраняются только в браузере текущего устройства.

## Локальный backend

В репозитории подготовлен бесплатный backend без внешних сервисов:

- публичный режим без регистрации;
- публичные точки в общей локальной базе;
- входящие и исходящие заявки на обмен;
- атомарное принятие обмена со сменой координат;
- обновление точек и заявок через регулярный polling;
- ежедневная очистка точек.

Данные не коммитятся в git: папка `data/` предназначена для локального хранения на машине, где запущен сервер.

`config.js` настроен так:

```js
// localhost / 192.168.x.x -> локальный same-origin API
// goswitch.ru -> https://api.goswitch.ru
```

Для публичного фронта нужен backend-адрес `https://api.goswitch.ru`.

## Локальный запуск

Из папки `ChangePlace`:

```powershell
npm start
```

Затем открыть:

```text
http://localhost:4173/
```

Service worker и установка на главный экран работают только через HTTP/HTTPS, поэтому для PWA-проверки лучше запускать через локальный сервер, а не открывать `index.html` напрямую.

## Публичная схема

Целевая схема:

```text
https://goswitch.ru
  GitHub Pages: HTML/CSS/JS/PWA

https://api.goswitch.ru
  этот ПК: Node.js API + data/changeplace-data.json
```

Node.js-сервер приложения слушает порт `4173`. Для HTTPS на поддомене `api.goswitch.ru` подготовлен `Caddyfile`:

```text
api.goswitch.ru {
  reverse_proxy 127.0.0.1:4173
}
```

Чтобы backend стал доступен из интернета:

1. В DNS добавить запись `A` для `api.goswitch.ru` на внешний IP этого ПК/роутера.
2. На роутере пробросить порты `80` и `443` на локальный IP этого ПК.
3. На Windows разрешить входящие подключения на `80` и `443`.
4. Запустить Node.js backend: `npm start`.
5. Запустить Caddy из папки проекта: `caddy run`.
